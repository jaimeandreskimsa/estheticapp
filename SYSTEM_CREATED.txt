🚀 INICIO RÁPIDO (5 minutos)

1. Instalar requisitos
   $ brew install node docker docker-compose  # macOS
   $ apt install node docker docker-compose   # Linux

2. Copiar variables de entorno
   $ cp .env.example .env
   $ nano .env  # Agregar OPENAI_API_KEY=sk-...

3. Iniciar servicios
   $ docker-compose up -d
   $ sleep 30  # Esperar que PostgreSQL esté listo

4. Ver aplicación
   Frontend: http://localhost:3000
   Backend:  http://localhost:3001
   Docs:     http://localhost:3001/api-docs

¡Eso es todo! El sistema está corriendo.

📋 LOS 4 MÓDULOS PRINCIPALES

Módulo 1: CONSULTA
  Captura: Motivo, Anamnesis, Evaluación Física, Fotos
  IA: Transcripción voz, Autocompletado anamnesis, Validación expectativas
  Storage: En clinical_records + photos

Módulo 2: DIAGNÓSTICO
  Input: Todo lo capturado en Módulo 1
  IA: GPT-4 genera diagnóstico clínico + resumen paciente + visual
  Output: Editable por médico (nunca impositivo)
  Storage: En diagnoses table

Módulo 3: PLAN DE TRATAMIENTO
  Input: Diagnóstico
  IA: Sugiere procedimientos, calcula cantidades, valida contraindicaciones
  Output: Plan estructurado + timeline visual + procedimientos ordenados
  Storage: En treatment_plans + treatment_plan_items

Módulo 4: PRESUPUESTO
  Input: Plan + Inventario de productos
  Lógica: Automática
    1. Por cada procedimiento: buscar producto
    2. Aplicar precio unitario
    3. Multiplicar por cantidad y sesiones
    4. Sumar total
    5. Calcular margen profesional
  Output: Presupuesto + versión paciente (sin costos internos)
  Storage: En budgets + budget_items

Flujo posterior:
  ↓ (si paciente acepta presupuesto)
  Consentimientos → Sesiones → Evolución → Historial

🗄️ BASE DE DATOS (PostgreSQL)

Entidades principales:
  Users                    - Profesionales médicos
  Clinics                  - Centros estéticos
  Patients                 - Pacientes
  ClinicalRecords          - Fichas clínicas
  Anamnesis                - Historial médico/estético
  PhysicalExaminations     - Evaluación física
  Diagnoses                - Diagnósticos
  TreatmentPlans           - Planes de tratamiento
  TreatmentPlanItems       - Procedimientos específicos
  Budgets                  - Presupuestos
  BudgetItems              - Items desglosados
  Consents                 - Consentimientos digitales
  Sessions                 - Sesiones de tratamiento
  SessionProcedures        - Procedimientos ejecutados
  ClinicalPhotos           - Fotos clínicas
  PostTreatmentInstructions - Indicaciones post-tx
  ClinicalEvolutions       - Seguimiento/evolución
  Products                 - Inventario
  ProductLots              - Control de lotes/vencimientos
  Payments                 - Cobranzas
  AuditLogs                - Trazabilidad legal

Relaciones:
  1 Clinic → N Users (doctores)
  1 Clinic → N Patients
  1 Patient → N ClinicalRecords
  1 ClinicalRecord → 1 Diagnosis, 1 Plan, 1 Budget, N Sessions
  1 TreatmentPlan → N TreatmentPlanItems
  1 Budget → N BudgetItems
  1 Session → N SessionProcedures
  etc.

Capacidad estimada:
  • 10.000 pacientes activos
  • 100.000 consultas/año
  • 1.000.000 fotos almacenadas
  • Queries <100ms con índices
